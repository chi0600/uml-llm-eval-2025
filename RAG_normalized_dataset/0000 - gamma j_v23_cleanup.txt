**Title:** Edit Customer Details – Edit Account

**Raw:**

Requirements:  
Edit Customer Details

Goal:  
Edit the customer account details.

Actors:  
Customer  
Weborder System

Preconditions:  
Customer must be logged-in on the system.

Triggers:  
Customer clicks button or link to “Edit Account”

Basic Scenario:  
1. Customer clicks button or link to initiate the process to edit the account.  
2. System displays account home page to Customer.  
3. Customer clicks button or link to edit the account details.  
4. System verifies the changes.  
5. System stores new account information.

Alternative Scenario:  
None

Postconditions:  
The Customer has changed the account details.

---

**Clean:**  
**Actors:** Customer, Weborder System  
**Preconditions:** Customer is already logged in to the system.  
**Triggers:** Customer clicks the **Edit Account** button or link.  

**Flow:**  
1. Customer → Weborder System: Click “Edit Account”  
2. Weborder System → Customer: Display account home page  
3. Customer → Weborder System: Click “Edit Account Details”  
4. Weborder System → Weborder System: Verify changes  
5. Weborder System → Weborder System: Store new account information  

**Alternative Scenarios:**  
- None

**Postconditions:**  
Customer account details are updated and persisted.
