**Title:** Install Plug-ins – Installation Process

**Raw:**

Requirements:  
Install Plug-ins

Goal:  
Install a new plug-in to the application.

Actors:  
Administrator  
Weborder System

Preconditions:  
Administrator must be able to access the weborder system via a web browser with HTTPS.

Triggers:  
Administrator clicks the button or link to “Install Plug-ins”

Basic Scenario:  
1. Administrator clicks the button or link to initiate plug-in installation process.  
2. System prompts the Administrator to upload the plug-in module.  
3. System installs the plug-in and validates changes.  
4. System displays plug-in options to the Administrator.

Postconditions:  
A new plug-in is installed in the application.

---

**Clean:**  
**Actors:** Administrator, Weborder System  
**Preconditions:** Administrator can access the Weborder System via HTTPS.  
**Triggers:** Administrator clicks the **Install Plug-ins** button or link.  

**Flow:**  
1. Administrator → Weborder System: Click “Install Plug-ins”  
2. Weborder System → Administrator: Prompt to upload plug-in module  
3. Administrator → Weborder System: Upload plug-in module  
4. Weborder System → Weborder System: Install and validate plug-in  
5. Weborder System → Administrator: Display plug-in options  

**Alternative Scenarios:**  
- None

**Postconditions:**  
Plug-in is installed and available in the system.
